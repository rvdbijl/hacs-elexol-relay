# Elexol Relay v0.1.0

Initial HACS-compatible release.

## Added

- UI setup for Elexol EtherIO host, UDP port, and available relay ports.
- Switch entities for each relay on selected ports.
- Event-driven UDP command sending when relay entities change state.
- Optimistic state restore without startup retransmission.
- HACS metadata and validation workflow.
- Tag-based GitHub Release workflow so HACS can discover semantic versions.

## Notes

Only one controller should send UDP commands to the same Elexol board. Disable existing Node-RED or other UDP command flows before making this integration the production controller.
