# Elexol Relay v0.2.0

## Added

- Readback support for Elexol Ether I/O 24 R ports using lowercase read commands `a`, `b`, and `c`.
- Startup state initialization from the relay board instead of Home Assistant restored state.
- Write verification: each relay change is followed by a readback of the affected port.
- Configurable readback polling interval. The default is 30 seconds; set it to 0 to disable background polling.
- Switch diagnostic attributes for port value, readback verification status, and latest communication error.

## Changed

- Switch states now represent the last value read from the relay board.
- Entities are marked unavailable when the board cannot be read.

## Notes

Readback confirms the Elexol output port value. It does not prove that a damper physically moved or that downstream equipment changed state.
